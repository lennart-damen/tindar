the Tindar Challenge

Challenge

Suppose you owned a dating company. Once every week, people come to your venue to find their ideal partner. During the speed-dating round, every person expresses if he/she would like to meet his/her date in real-life in the next week. Your people have busy schedules, so they can only go on a real date with a single person.

Your job is to make your community as happy as possible by pairing up as many users as you can. You can only pair people if there is mutual interest. Tindar is monogamous: no threesomes, and self-love doesn't count.

Solution

As explained in the Github documentation, the Tindar community can be represented as a graph with the people as nodes and the edges for interest. This problem is then readily defined as a Binary Linear Problem, which can theoretically be solved with algorithms such as Branch and Bound. In practice, we use the PuLP library which does the optimisation for us. I have also developed a heuristic as a baseline to compare PuLP against.

Web Components

Apart from this homepage, you can visit:

  • .../api
    for explanation on how to generate and solve Tindar problems

  • .../api/generate
    by making a GET request with a query string.
    You will receive a JSON file back with a randomly generated Tindar problem and some extra parameters.*

  • .../api/solve
    by making a JSON POST request of a Tindar problem. You will receive a JSON file back with the solution to the Tindar problem that you posted.*

  • .../report, make a GET request to get the Tindar report as PDF with a full explanation on the Tindar challenge, the optimisation model, the PuLP and heuristic solutions, a comparison of their performance, and an overview of the deployment process.

*If you make a simple GET request, you will receive the same HTML as .../api.
