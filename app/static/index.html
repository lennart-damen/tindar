the Tindar Challenge

Challenge

Suppose you owned a dating company. Once every week, people come to your venue to find their ideal partner. During the speed-dating round, every person expresses if he/she would like to meet his/her date in real-life in the next week. Your people have busy schedules, so they can only go on a real date with a single person.

Your job is to make your community as happy as possible by pairing up as many users as you can.

Solution

As explained in the Github documentation, the Tindar community can be represented as a graph with the people as nodes and the edges for interest. This problem is then readily defined as a Binary Linear Problem, which can theoretically be solved with algorithms such as Branch and Bound. In practice, we use the Pulp library which does the optimisation for us.

Web Components

Apart from this homepage, you can visit:

  • .../api
    for explanation on how to generate and solve Tindar problems

  • .../api/generate
    by making a GET request with a query string.
    You will receive a JSON file back with a randomly generated problem and some extra parameters

  • .../api/solve
    by making a JSON POST request. You will receive a JSON file back with the solution to the Tindar problem that you posted.
